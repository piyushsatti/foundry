Dev-machine setup tasks, bugs, and resolutions are tracked in `/home/piyush/Documents/notes/journal/dev-machine-problems.md`.

**Format:** Markdown checklist. Open items at the top under `### Claude Code Setup`, closed items at the bottom. Each entry is a single bulleted line: `- [ ]` or `- [x]` + **bold title** + em-dash + description + *(date)* in italics. Closed items include resolution details and dates in `YYYY-MM-DD` format.

**How to apply:** When work in a session changes the dev-machine state (Claude Code config, statusline, hooks, system services, dotfiles, audits), append or update the corresponding entry. When opening the file fresh, it doubles as a backlog — prompt the user to pick an open item if they're looking for something to do.
