Document everything worthwhile; remember only what repeats.

**Why:** Piyush's framing — memory is a subset of documentation, not an alternative to it. Worthwhile one-offs still get documented (journal, project notes, inline). Recurring patterns get documented AND saved to memory so future sessions surface them automatically. Filtering by "ephemeral vs eternal" or "specific vs generic" is the wrong axis — recurrence is.

**How to apply:** Before saving memory, ask two questions in order:
1. Is this worth documenting at all? If no, drop it.
2. Will I encounter it repeatedly across future sessions? If yes, also save to memory (and ideally point the memory at the document). If no, document only.

When proposing a memory save to Piyush, include the evaluation against these two gates so he can make an informed call.
